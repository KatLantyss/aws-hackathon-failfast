# 維修效益分析 — 為什麼 FOC 有時上升？

## 🔴 問題現象

你看到的前端數據中，有**大量維修後 FOC 上升**的案例：

```
S1 Day 981 DD：    39.67 → 50.96 (+28.5%)  ❌
S2 Day 985 DD：    14.78 → 28.85 (+95%)   ❌❌
S2 Day 1218：      42.07 → 61.59 (+46%)   ❌
```

**直覺**：維修（清洗、拋光）應該**減少**阻力 → 油耗下降

**實際**：油耗反而上升 → 維修失敗？

---

## 🔍 根本原因：30 天平均的陷阱

### 問題在於「運營模式改變」

維修前 30 天 vs 維修後 30 天，可能是完全不同的運營場景：

```
維修前 30 天：
  - 船可能因為汙損嚴重而被迫低速
  - 平均 RPM = 40
  - 平均 FOC = 39.67 MT/day（低速）

維修後 30 天：
  - 維修後恢復高速（運營需求）
  - 平均 RPM = 75
  - 平均 FOC = 50.96 MT/day（高速）

結果：
  ❌ 同樣 RPM 下無法對比
  ❌ 高 RPM 自然油耗高
  ❌ 看不出「相同運營條件下」的改善
```

### 類比

```
你的車：
  汙積了 1 年沒洗
  被迫開 40 km/h（怕壞）→ 油耗 5 L/100km

洗車後：
  恢復正常
  改為開 100 km/h（恢復日常）→ 油耗 8 L/100km

結果：「洗車後油耗反而升高！」❌（錯誤對比）
正確對比：「同樣 100 km/h，洗車前是 12 L/100km，洗車後是 8 L/100km」✓
```

---

## ✅ 正確的維修效益指標

### 方案 1：**相同 RPM 下的 STW 對比**（最精確）

```
維修前：RPM = 75 時，STW = 18.0 knots
維修後：RPM = 75 時，STW = 18.5 knots （⬆️ 改善）

Speed Loss 下降 ✓
```

**優點**：隔離 RPM 因素，只看效能本身

### 方案 2：**運營效率比（噸英里油耗）**

```
燃油效率 = (Speed × Cargo) / FOC

如果維修後在高 RPM 下：
  - FOC ⬆️ 上升（因為高速）
  - 但 Speed ⬆️ 也上升
  - 最終「每噸每海里」的油耗可能 ⬇️ 下降
```

### 方案 3：**控制變數法**（實務）

```
選擇「維修前後都有相同 RPM 帶的數據」進行對比

例如：
  Day 975-980（維修前）: RPM = 75 → STW = 18.0, FOC = 50
  Day 985-990（維修後）: RPM = 75 → STW = 18.8, FOC = 48
  
  結論：✓ 相同速度下，油耗實際下降
```

```
結論：不是維修失敗，是運營模式改變
```

## 💡 真實的維修信號

| 指標 | 解讀 |
|------|------|
| **相同 RPM 下 STW ⬆️** | ✅ 效能真的恢復了 |
| **相同 RPM 下 FOC ⬇️** | ✅ 真正省油了 |
| **相同 Speed 下 RPM ⬇️** | ✅ 效能提升，可以用更低轉速跑 |
| **30 天平均 FOC ⬆️** | ⚠️ 可能只是因為跑高速了，看 RPM 才能判斷 |
