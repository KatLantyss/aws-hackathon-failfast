# Theme.md — 品牌配色系統（取代 Navy + Amber 通用樣板）

> 給 Kiro / Claude Code 的配色升級指令文件
> 版本：v1.0　最後更新：2026-07-10
> 取代對象：《Frontend Design Requirements》第 3.2 節色彩系統
> 依據：陽明海運（Yang Ming）品牌識別素材（紅／黑／白為主視覺）

---

## 0. 為什麼要改：問題不是「不好看」，是「跟品牌無關」

目前系統的 navy（深藍）+ brass-amber（金色）配色本身沒有做錯，但問題在於：**這套配色是通用的「航海儀表美學樣板」，跟陽明海運自己的品牌識別完全無關**，感覺像任何一家航運公司都能套用的系統，缺乏「這是我們公司的系統」的識別感。

陽明的品牌識別非常明確：**強烈的品牌紅、黑、白**，加上乾淨有力的粗體大寫字體。這份文件的目標是把這個真實品牌資產導入系統，同時解決一個關鍵風險：**品牌紅不能跟既有的警示紅語意混淆**。

> **重要聲明（版權/素材使用範圍）**：本文件僅提取品牌識別素材中的色彩數值與使用邏輯，不複製、不重製任何海報上的插畫、標語或版面設計。實際的精確品牌色號，請以貴公司內部品牌識別手冊（Brand Guideline）為準,下方 Hex 為依素材色感估算的建議值，正式上線前請對照官方 Pantone/Hex 色卡校正。

## 1. 核心原則：品牌色與功能色分工

這是整份文件最重要的規則，所有色彩決策都從這裡出發：

| 角色 | 定義 | 舉例 |
|---|---|---|
| **品牌識別色（Brand Identity）** | 代表「這是陽明的系統」，用在品牌會被看到的地方 | 頁首 Logo 區、主要 CTA 按鈕、目前選中的導航項目、signature 元件的強調線 |
| **功能狀態色（Functional Status）** | 代表資料的意義（正常/警告/危險），與品牌色**必須可明確區分**，不可共用同一個紅 | 測深計量表的風險分級、船況狀態燈、異常值標示 |

**規則**：品牌紅只出現在「識別/操作」語境，不出現在「資料狀態判讀」語境。兩者的色相/飽和度要刻意錯開,讓使用者一眼就能分辨「這是品牌裝飾」還是「這是真的要注意的警示」。

## 2. 色彩 Token 定義（含遷移對照表）

**實作策略：保留既有 CSS 變數「名稱」，只更新色值，新增品牌色 token。** 這樣既有元件（`FathometerGauge.vue`、`PanelTag.vue`、`KpiCard.vue` 等）不需要逐一修改程式碼裡的 class 或 binding，只要更新 `:root` 的色值定義即可全站套用，大幅降低回歸風險。

| Token 名稱 | 舊值（v1.0） | 新值（v2.0） | 角色 | 說明 |
|---|---|---|---|---|
| `--color-brand-red`（**新增**） | 無 | `#D7191F`（建議值，需對照官方色卡校正） | 品牌識別 | 陽明品牌紅,系統中最具識別性的顏色 |
| `--color-marine-navy` | `#0B1F33` | `#181818`（近黑，非純黑） | 品牌識別（頁首/側邊導航底色） | 從「深藍」改為「近黑」，呼應品牌黑，同時脫離「navy」這個被認為制式的顏色本身 |
| `--color-chart-paper` | `#EDE6D6`（暖米色) | `#F2F0ED`（極淺冷灰白） | 中性底色 | 從偏黃暖色調整為更中性的冷灰白，避免跟紅色搭配時顯得像賀卡而非專業系統 |
| `--color-brass-amber` | `#C08A3E` | `#C08A3E`（不變，但**角色改變**） | 功能狀態色（僅警告語意） | 不再是「主要裝飾強調色」，只在「中等信心/警告」這種功能語意下使用，例如信心分級標籤 |
| `--color-signal-red` | `#C1443C` | `#E2572B`（暖橘紅/朱紅） | 功能狀態色（僅危險語意） | **刻意調整為偏橘的紅**，與品牌紅的色相拉開差距，避免使用者混淆「品牌色」跟「危險警示」 |
| `--color-fathom-teal` | `#3E7C74` | `#3E7C74`（不變） | 功能狀態色（正常/健康語意） | 未受品牌色調整影響，繼續沿用 |
| `--color-ink-slate` | `#28313C` | `#26262A`（微調更中性） | 文字/線條 | 跟新的近黑品牌黑更協調 |

### 2.1 氛圍限定色（第二份升級文件已定義，維持不變)

`--atmo-bioluminescent`、`--atmo-abyss-violet` 兩色維持原定義,僅用於登入畫面/頁首粒子效果，與本次品牌色調整互不影響（不同使用場景,不會同時出現在同一視野內造成衝突）。

## 3. 品牌紅的具體使用範圍

| 使用位置 | 是否套用品牌紅 | 說明 |
|---|---|---|
| 頁首 Logo 標記 / 系統名稱旁的識別線 | ✅ | 品牌露出的核心位置 |
| 主要 CTA 按鈕（如「查看完整頁面」「儲存」等主要動作按鈕） | ✅ | 取代原本 brass-amber 的主要按鈕角色 |
| 目前選中的導航項目底色/底線 | ✅ | 取代原本 brass-amber 的選中狀態 |
| 雷達/聲測環的掃描線顏色（第二份文件 6.3 節） | ✅（由 brass-amber 改為 brand-red） | 這是裝飾性 signature 元素，屬於識別語境,不是資料狀態,適合換成品牌紅 |
| 測深計量表指針顏色 | ❌ 維持功能色（依風險程度呈現 teal/amber/signal-red 分級），不套用品牌紅 | 指針顏色本身就是資料語意，不能被品牌識別色取代 |
| 船況狀態燈（航行中/靠泊中/錨泊中） | ❌ | 維持功能色系統，與品牌色無關 |
| Speed Loss 圖表中的異常標示 | ❌ | 維持 `--color-signal-red`（新的暖橘紅），不可用品牌紅，否則會讓使用者誤以為異常標示只是裝飾 |

## 4. 中性底色調整說明

原本的 `--color-chart-paper` 偏暖米色（航海圖紙意象），單獨使用沒問題，但**跟高飽和度的品牌紅放在一起時容易顯得像賀卡或促銷海報，而非專業維運系統**。調整為更中性、更冷調的淺灰白，讓紅色作為強調色時更「乾淨」、更有科技/工業感，同時仍保留「紙面」的柔和質感，不會變成純白的冷硬感。

---

## 5. 字體調整建議（次要，若 Kiro 有餘力可一併處理）

陽明品牌素材中的標題字體是**粗體、大寫、稍微傾斜/動態感的無襯線字體**。目前系統 Display 字體（Oswald/Bebas Neue 窄體大寫）方向已經相符，**不需要更換字體家族**，但可以：
- 系統中「品牌識別語境」的標題（如頁首系統名稱、登入畫面標題）可以加大字重至 Bold/Black，呼應品牌海報標題的力量感
- 一般資料頁面標題維持現有字重，不需要全站都加粗,避免資料頁面顯得過於用力

## 6. 給 Kiro 的具體執行指令（可直接複製使用）

建議你貼給 Kiro 的指令長這樣，重點是**明確要求「只改色值,不要重寫元件邏輯」**，避免它把已經做好的功能又重新生成一次：

```
請依照 theme.md 更新全站配色，注意以下限制：

1. 只修改 CSS 變數（design tokens）的色值定義，不要修改任何 .vue 元件的
   結構、邏輯或 class binding。所有元件應該透過既有的 var(--color-xxx)
   自動套用新色值。

2. 新增 --color-brand-red 這個 token，並依 theme.md 第 3 節的使用範圍表,
   把目前用 brass-amber 做「主要按鈕/選中狀態/雷達掃描線」的地方改成引用
   brand-red，其餘 brass-amber 的使用（信心分級等功能語意）維持不變。

3. --color-signal-red 的色值請更新為新的暖橘紅（見 theme.md 第2節表格），
   這是為了跟品牌紅做出區隔，避免使用者混淆「品牌識別」與「危險警示」。

4. 完成後請提供 Before/After 截圖，讓我確認品牌紅有沒有不小心用到資料
   狀態判讀的地方（例如測深計指針、船況狀態燈、異常標示），這些地方
   規則上不可以用品牌紅。

5. 若程式碼中有直接寫死 hex 色碼（沒有透過 CSS 變數的），請列出清單給我,
   我們之後統一處理成變數,不要在這次順手改掉,避免範圍擴大。
```
