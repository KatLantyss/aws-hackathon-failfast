### 場景 2️⃣：中度污損 + 船殼為主 → UWC 建議

**前置條件**：
- 船舶：S2
- 當前 Speed Loss：22%
- Hull 污損：75%，Prop 污損：25%
- 衰退速率：0.08%/日
- 燃油消耗：50 MT/日

**預期行為**：

| 元素 | 預期值 | 驗證方法 |
|------|--------|--------|
| 緊急程度 | ⚠️ HIGH (橙色) | 檢查卡片邊框 |
| 建議 | 船殼清洗 (UWC) | 根因分析：Hull > 70% |
| 成功率 | 16.7% ⚠️ | 檢查預期效益 |
| 估計成本 | $45K | 最便宜的選項 |
| 根因（Hull） | 75% | 進度條 > 70% →  UWC |
| 根因（Prop） | 25% | 進度條 < 70% →  不推薦 PP |
| **ROI** | 應該可計算 | 檢查面板 |
| 月度節省 | 應 > $1,000 | 相對較低（UWC 成功率低） |
| 預警 | 提示"成功率較低" | 檢查理由文字 |

**驗證點**：
- ✅ 根因分析正確：Hull > 70% → UWC 而非 UWC+PP
- ✅ UWC 成功率低被準確反映在 ROI 中（月度節省相對低）
- ✅ 即使成功率低，仍建議 UWC 而非 PP（因為 Hull 污損為主）

## 📊 ROI 面板測試

### 驗證數據源

```
✅ 應該從 fuelPrediction.value.counterfactual_uwc_pp 獲取：
- fuel_saving_mt_per_day
- saving_pct
- energy_pricing.daily_saving_usd

❌ 不應該使用舊公式 (latestSL × 0.5)
```

### 驗證計算

```
月度節省 = dailySavingUsd × 30 × (successRate / 100)
年度節省 = 月度節省 × 12
回本週期 = estimatedCost / 月度節省
ROI% = ((年度節省 - estimatedCost/12) / estimatedCost) × 100
```

**測試用例**：
- dailySavingUsd = $150
- successRate = 57.1%
- estimatedCost = $78,000

**預期結果**：
- monthlySavings = 150 × 30 × 0.571 = $2,570
- annualSavings = $2,570 × 12 = $30,840
- paybackMonths = $78,000 / $2,570 ≈ 30.4 個月
- ROI% = (($30,840 - $6,500) / $78,000) × 100 ≈ 31%

## 🐛 邊界情況測試

### 邊界 1：燃油預測未加載

**情況**：API 遲到或失敗

**預期**：
- ROI 面板不顯示（null check）
- 頁面不崩潰
- 顯示錯誤訊息或空狀態

**驗證**：
```javascript
if (!fuelPrediction.value) return null
```

---

### 邊界 2：根因數據缺失

**情況**：slData.value.summary 為 null

**預期**：
- 使用預設值 hull_pct = 50%, prop_pct = 50%
- 推薦 UWC+PP（預設行為）
- 不崩潰

**驗證**：
```javascript
const hullPct = slData.value.summary?.hull_pct ?? 50  // ?? 50
```

**情況**：vessel.degradationRatePctPerDay = 0

**預期**：
- 不顯示時機預測（或提示"穩定"）
- 不計算 Math.ceil(Infinity)

**驗證**：
```javascript
if (vessel.degradationRatePctPerDay) { ... }
```
