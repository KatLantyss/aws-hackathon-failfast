## 📊 計算能力

### 數據規模
| 指標 | 數值 |
|------|------|
| 正午報表記錄 | 21,282 條 |
| 清洗後有效記錄 | 18,987 條 |
| 船舶數量 | 15 艘 |
| 時間跨度 | 5 年（2021-2025） |
| 維修事件 | 77 條 |
| 計算耗時 | 2-3 分鐘 |

### 計算公式
**Layer 1 (相對基準法):**
```
Speed Loss L1 = (Daily FOC - 該船歷史最佳 FOC) / 基準 FOC × 100%
```

**Layer 2 (多因素回歸法):**
```
Expected FOC = Baseline × (1 + 衰退 + 風力 + 季節 + 負載)
Speed Loss L2 = (Expected - Actual) / Expected × 100%
```

**Layer 3 (維修效益法):**
```
效益 = (維修前 FOC - 維修後 FOC) / 維修前 FOC × 100%
```

### speed_loss_output/

| 文件 | 大小 | 用途 | 行數 |
|------|------|------|------|
| `speed_loss_complete.csv` | 6.6 MB | 完整計算數據 | 18,987 |

| `visualization_data.json` | 1.9 MB | 前端可視化數據 | - |

### 關鍵欄位說明

**speed_loss_complete.csv:**
- `ship_id` — 船舶代號
- `NOON_UTC` — 日期（相對天數）
- `daily_foc` — 該日油耗（MT/day）
- `speed_loss_pct_l1` — 效能損失（L1）
- `speed_loss_pct_l2` — 效能損失（L2）
- `expected_foc` — 多因素模型預期油耗
- `days_since_maintenance` — 距上次維修天數
- `maintenance_event` — 維修事件類型
- `maintenance_improvement_pct` — 維修效益（%）
