## 📊 方法對比

### 中位數差值法（舊方法）

```
維修前 Speed Loss：[10%, 12%, 15%, 18%, 20%]
  中位數 = 15%

維修後 Speed Loss：[8%, 10%, 12%, 14%, 16%]
  中位數 = 12%

結論：Δ = 12% - 15% = -3% ✓ 改善

問題：
❌ 只看單一時間點（中位數）
❌ 無法區分「突然改善」vs「逐漸改善」
❌ 對異常值敏感
❌ 無法捕捉「防止衰退」的效果
```

### 斜率變化法（新方法） ✅

```
維修前 Speed Loss 趨勢（線性回歸）：
  每天增加 +0.050 %/day
  （Speed Loss 在衰退）

維修後 Speed Loss 趨勢（線性回歸）：
  每天減少 -0.020 %/day
  （Speed Loss 在改善）

結論：斜率改善 = 0.050 - (-0.020) = 0.070 %/day
      ✅ 優秀改善！

優勢：
✅ 看完整趨勢，不只看單點
✅ 能區分維修效果（進塢 > 清洗）
✅ 對異常值不敏感（用中值濾波）
✅ 直接反映「維修阻止了衰退」
✅ 更符合航海邏輯
```
